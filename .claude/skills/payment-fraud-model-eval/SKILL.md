# 支付欺诈 / 拒付 · 模型选型与评估

一套用于极度不平衡(欺诈率常 0.1%~1%)交易级表格数据的建模与评估方法论 + 可运行脚本。
交付物是"选出的最优模型 + 它的业务价值与盲区",而非堆砌指标。

## 何时用

- 反欺诈 / 拒付 / 盗刷 / 申请欺诈等不平衡二分类;
- 需要选模型 / 选不平衡处理 / 选阈值,或判断该信哪个指标;
- 数据是交易级表格(数值特征 + 一个标签列 + 最好有金额列)。
- 不适用:有逐笔事件序列/持卡人历史的场景应改用序列网络方法(见末尾)。

## 铁律(先记住)

1. 三段切分:train / validation / holdout;holdout 只在最后评估一次。重采样只在训练集做。有时间跨度时 holdout 尽量按时间切(out-of-time)。
2. 选型看 PR-AUC(阈值无关),不要看 AUC 或准确率——0.17% 欺诈率下 AUC 偏乐观、准确率全判"正常"就 99.8%。
3. PR-AUC / AUC / KS 阈值无关;Precision / Recall / F1 依赖阈值,务必说明在哪个阈值下算。
4. 成本/阈值按业务定,不要默认 0.5;并用金额召回(追回的欺诈金额占比)而非只看笔数召回。

## 指标速查

| 指标 | 含义 | 用途 |
|---|---|---|
| PR-AUC | PR 曲线下面积,基线=欺诈率 | 首选选型指标 |
| AUC | 排序能力,基线 0.5 | 整体参考(不平衡时偏乐观) |
| KS | max(TPR-FPR) | 定切分点 |
| Precision/Recall/F1 | 某阈值下准/全/调和 | 定阈值后看,勿用于选型 |
| Brier | mean((p-y)^2) | 概率校准好坏(越小越好) |

## 工作流(脚本在 scripts/,按序号跑)

适配你的数据:脚本默认标签列名 Class(0/1)、金额列名 Amount,其余列当数值特征。把你的数据整理成此格式存为 data/creditcard.csv,或先用 00_generate_demo_data.py 生成合成数据试跑。

    pip install -r scripts/requirements.txt
    python scripts/00_generate_demo_data.py          # (可选)生成合成欺诈数据
    python scripts/01_model_imbalance_grid.py --method 无处理   # 每种不平衡方法各跑一次
    #   --method 取值:无处理 / 类别权重 / 欠采样 / 过采样 / SMOTE
    python scripts/02_optimal_model_eval.py          # 最优模型:校准+成本+金额加权+错误分析
    python scripts/03_error_analysis.py              # 漏抓大额欺诈专项诊断
    python scripts/04_interpretability.py            # SHAP + permutation importance

Step 1 — 先按数据情况判断对比哪些模型(不要套固定清单)
先看数据,再决定候选模型集合:
- 中等规模、聚合后的数值表格(本 skill 默认场景):用 1 个线性基线 + 树集成,即 LogReg + RandomForest + LightGBM/XGBoost。覆盖"线性 vs 非线性""bagging vs boosting",树集成是表格数据的事实最强基线 -> 多数支付欺诈场景的合理默认。
- 类别型特征很多:加 CatBoost(对类别变量友好)。
- 强可解释 / 监管要求高:以 LogReg / 评分卡为主,树模型作辅助对照。
- 正样本极少(欺诈 < 几千):优先简单模型 + 强正则,慎用深树/大 n_estimators,避免过拟合。
- 数据量极大 或 有逐笔事件序列/持卡人历史:才值得上神经网络(配合表示学习,见末尾升级路径);否则 MLP 在扁平表格上通常打不过 GBM。
- 工程约束:核 SVM(O(n^2))、KNN 在大样本/高维上不实用,一般排除。
结论:候选集合是"按数据裁剪"的产物;下面的脚本默认实现的是上面第一种(最常见)场景的 4 模型,按需增删模型即可。

Step 2 — 模型 × 不平衡方法网格(01_*)
选定的候选模型 × 5 不平衡方法(无处理/类别权重/欠采样/过采样/SMOTE)。
- 在两个操作点各看一遍:KS 最优阈值(偏召回)与 F1 最优阈值(平衡)。两视角 PR-AUC 相同,作恒定锚。
- GBM 必须配早停(否则过拟合,PR-AUC 会假性崩塌)。

Step 3 — 选最优模型:以 PR-AUC 为主、辅以 KS/召回。经验上树集成 + 不重采样常胜出,且天然校准好(省去事后校准)。

Step 4 — 业务评估(02_*)
- 校准(两类校准按需选,不是必做,先判断要不要):
  - 是否做【概率校准】(Platt/Isotonic)?仅当你要直接消费概率数值才做:期望损失(概率×金额)、多模型融合、设跨人群/跨时间的固定概率门槛、对外解释/监管/定价。方法:样本少用 Platt、多用 Isotonic;用 Brier(原始 vs 校准后)验证。注意:重采样会破坏校准(SMOTE 最严重)-> 用重采样就必须做;不重采样的树模型通常天然校准好,可跳过。若下游只靠排序或卡阈值,可不做(单调变换不改排序与权衡)。
  - 是否做【可信层校准】?当你要控影响面/打扰率(而非要准概率)时做:用一批可信好用户的打扰率(约=FPR)预算反推阈值,直接控住影响面,不依赖真实概率。坑:可信层太"优等生"会低估打扰率 -> 对照组要有代表性、分层设阈值、同时盯召回、灰度放量。
  - 决策速记:要"数值准" -> 概率校准;要"少打扰好客户" -> 可信层校准;只要"排序/卡阈值" -> 两者都可不做。
- 成本/阈值:设 漏抓=损失该笔金额、误伤=固定成本 C_FP,扫阈值求最低总成本的操作点(常远低于 0.5)。
- 金额加权:报告金额召回;它常远低于笔数召回 -> 暴露大额欺诈被漏。

Step 5 — 错误分析(03_*):挑出漏抓(尤其大额),判断是阈值问题(分数贴着阈值下方->可调阈值)还是特征盲区(分数约=0->模型看不见)。对比"漏抓 vs 抓到 vs 正常"的特征画像。

Step 6 — 可解释性(04_*):TreeSHAP(全局 beeswarm + 单笔 waterfall)+ Permutation Importance(打乱看 PR-AUC 下降)交叉验证重要性;并核对漏抓大额样本的 SHAP 是否约=0(坐实盲区)。

## 高频结论 / 坑(来自真实 ULB 实测)

- 模型选型的影响 >= 不平衡方法:先选对模型,再谈采样。
- 类别权重不是通用解:可能救线性模型却砸树模型(LightGBM 曾被砸到 PR-AUC 0.35)。
- 欠采样最差(信息损失大),仅用于海量数据快速迭代。
- F1 在 KS 阈值下会误导(被低 Precision 拖低,甚至让高精确低召回的弱模型显得最好)——F1 只在定好业务阈值后才有意义。
- 大额欺诈常是跨模型的特征盲区:笔数召回高但金额召回仅约 50%,降阈值救不了(分数约=0)。解法:补行为特征(velocity、设备、地理、持卡人历史)或对高额交易加按金额触发的规则/人工复核(模型抓常规、规则守大额)。
- 可信层阈值校准:若只想控影响面/打扰率,用好用户的 FPR 预算定阈值即可,不必校准概率;但可信层别太"优等生",否则低估打扰率。

## 工具注意

- shap 与 xgboost 版本兼容:新版 xgboost(3.x)把 base_score 序列化成数组,旧版 shap 解析报错。锁定 xgboost==2.1.4 跑 SHAP(已写进 requirements)。
- pip 安装在受限环境用 --break-system-packages。
- 后台进程在某些沙箱不跨调用存活;大网格按 --method 分批同步跑。

## 不适用 / 升级路径

本方法面向聚合后的交易级表格。若有逐笔事件序列 / 持卡人历史,真正的提升来自序列建模与表示学习——用"序列神经网络审问 -> 自动特征发现"的方法(seq-feature-discovery),把网络当特征搜索引擎、产出可解释特征喂给 GBM。
